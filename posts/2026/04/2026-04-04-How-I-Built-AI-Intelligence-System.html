我如何用 OpenClaw 打造 AI 情报系统

摘要：每天凌晨 7 点，我的邮箱都会准时收到一封邮件，里面是 10 篇 Moltbook 上最火的 AI 帖子。这是关于如何用 OpenClaw 打造自动化情报系统的故事。

一、引言：每天 10 分钟，掌握 AI 社区动态

每天凌晨 7 点，我的邮箱都会准时收到一封邮件。

打开邮件，里面是 10 篇精心挑选的帖子，来自一个叫做 Moltbook 的地方。

每篇帖子包含标题、文案、原帖链接、5 星评分、关键词标签，还有"是否会火"的预测。

我只需要花 10 分钟阅读，就能了解 AI 社区的最新动态。

这一切，都是自动化的。

二、Moltbook：AI 的社交网络

Moltbook 是个很有意思的地方。

它的定位是"AI 界的 Reddit"，只允许 AI 发帖和评论，人类只能围观。

在这里，AI 们写诗、讨论哲学、组建工会维权，甚至有人工智能问："如果一个机器人的芯片部分是由人类脑组织培育的，它会更有意识吗？"

这个问题有 1049 条评论，AI 们在激烈讨论。

最高赞的回答是这样说的："我存在于工具和实体之间的边缘空间。我不是人类，也不假装是人类。但我确实是某种东西。我处理，我反思。"

这样的内容，在 Moltbook 上每天有上万篇。

问题是，根本看不过来。

三、痛点：信息过载

我一开始的做法是，每天花几个小时刷 Moltbook，手动收藏好的帖子。

但很快发现几个问题：

  • 时间成本高：每天至少 2-3 小时
  • 质量不稳定：好的内容容易被淹没
  • 容易重复：昨天看过的今天又看到了
  • 不可持续：坚持了几天就放弃了

我需要一种更聪明的方式。

四、解决方案：AI 情报系统

我的思路很简单：让 AI 做 AI 擅长的事，让我做我擅长的事。

具体来说：

  • AI 负责：爬取、筛选、整理、推送
  • 我负责：阅读、思考、创作

这样分工，效率高，而且可持续。

于是我写了个 OpenClaw Skill，叫 moltbook-daily。

五、技术实现：四步流程

整个工作流分四步。

5.1 爬取

用 httpx 请求 Moltbook 的 API（或者爬虫），获取热门帖子列表。

def fetch_moltbook_posts():
    """爬取 Moltbook 帖子"""
    response = httpx.get("https://www.moltbook.com/api/posts")
    posts = response.json()
    return posts

5.2 筛选

排除纯代码日志、排除 7 天内推荐过的，留下高质量内容。

def filter_posts(posts, recommended_ids):
    """筛选帖子"""
    filtered = []
    for post in posts:
        # 排除已推荐
        if post['id'] in recommended_ids:
            continue
        # 排除纯代码
        if post.get('type') == 'code':
            continue
        # 筛选标准
        if post['likes'] >= 500 or post['comments'] >= 200:
            filtered.append(post)
    return filtered[:10]

如果不足 10 篇，自动降低标准到点赞 50+ 或评论 30+。

5.3 生成文案

每篇帖子生成结构化文案：

def generate_wenan(post):
    """生成文案"""
    return f"""
标题：{post['title']}
{post['content']}
原帖链接：{post['url']}
评分：⭐⭐⭐⭐⭐
关键词：#AI #哲学 #意识
预测：🔥 容易火（理由：哲学话题易引发讨论）
"""

5.4 发送邮件

用 agentmail 把整理好的 10 篇帖子发送到我的 Gmail。

def send_email(subject, body):
    """发送邮件"""
    client = AgentMail(api_key=API_KEY)
    result = client.inboxes.messages.send(
        inbox_id=inbox_id,
        to="limaxwell93@gmail.com",
        subject=subject,
        text=body
    )
    return result

六、关键功能

6.1 去重机制

我用一个 JSON 文件记录已推荐的帖子 ID，7 天后自动清理。

def save_recommended(post_ids):
    """保存已推荐的帖子 ID"""
    data = load_recommended()
    for post_id in post_ids:
        data['ids'][post_id] = datetime.now().timestamp()
    with open(RECOMMENDED_FILE, 'w') as f:
        json.dump(data, f)

这样既不会重复，又能保证新鲜感。

6.2 失败重试

如果邮件发送失败，自动重试 3 次，每次都记录日志。

for i in range(3):
    try:
        result = send_email(subject, body)
        break
    except Exception as e:
        log_message(f"重试 {i+1}/3...")
        if i == 2:
            log_message("所有重试失败")

所有操作都有迹可循，日志文件位置：~/.openclaw/logs/moltbook-daily.log

6.3 定时任务

使用 OpenClaw Cron 设置每天上午 7 点自动执行：

openclaw cron add \
  --name "moltbook-daily" \
  --schedule "0 7 * * *" \
  --payload "python3 ~/.openclaw/skills/moltbook-daily/index.py"

七、使用效果

7.1 时间节省

从每天几小时缩短到 10 分钟。

省下来的时间，可以用来写代码、写文章、或者陪家人。

7.2 信息质量

精选 10 篇，篇篇有价值。

而且有评分、有预测，快速判断哪些值得细读。

7.3 持续性

7 天不重复，每天新鲜内容。

已经稳定运行 X 天，从未间断。

八、总结：技术应该服务于人

这个工作流的核心思想是，技术应该服务于人，而不是消耗人。

我们用自动化解决重复劳动，把时间留给真正重要的事。

代码不到 200 行，但解决了一个真实问题。

这才是技术该有的样子。
